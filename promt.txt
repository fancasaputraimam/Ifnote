==================================================
STRICT EXECUTION RULES — WAJIB
==================================================

Create an auto installer script for deploying ifNote Web on a VPS.

Do NOT change the app source code.
Do NOT redesign UI.
Do NOT modify frontend/backend features.
Do NOT expose API keys.
Do NOT hardcode secrets.
Do NOT remove existing env logic.
Only create deployment/installer files.

Target VPS:
- Ubuntu/Debian Linux
- Node.js app
- Frontend: Next.js
- Backend: NestJS/Express
- Database: PostgreSQL
- Process manager: PM2
- Reverse proxy: Nginx
- Optional SSL: Certbot

==================================================
MAIN GOAL
==================================================

Create an interactive VPS auto-install script that can install:

1. Backend only
2. Frontend only
3. Backend + Frontend

Database choices:

1. Install PostgreSQL directly on VPS
2. Use external PostgreSQL provider

If user chooses external PostgreSQL:
- ask for PostgreSQL credentials
- generate DATABASE_URL automatically
- support SSL mode

If user chooses local PostgreSQL:
- install PostgreSQL
- create database
- create database user
- create password
- generate DATABASE_URL automatically

==================================================
DATABASE OPTIONS
==================================================

If installing backend, ask database mode:

1. Install PostgreSQL on VPS
2. Use external PostgreSQL provider

LOCAL POSTGRES MODE:
- install postgresql and postgresql-contrib
- start and enable PostgreSQL
- ask:
  - database name, default ifnote
  - database user, default ifnote_user
  - database password
- create database and user
- grant privileges
- generate:

DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/DB_NAME?schema=public"

EXTERNAL POSTGRES MODE:
Ask:
- host
- port, default 5432
- database name
- username
- password
- ssl true/false, default true

Generate:

If SSL true:
DATABASE_URL="postgresql://USER:PASSWORD@HOST:PORT/DB_NAME?schema=public&sslmode=require"

If SSL false:
DATABASE_URL="postgresql://USER:PASSWORD@HOST:PORT/DB_NAME?schema=public"

==================================================
BACKEND INSTALL
==================================================

For backend installation:

1. Create backend .env.

Required env:

NODE_ENV=production
PORT=3001
DATABASE_URL=...
JWT_SECRET=generated_secure_secret
AI_API_KEY=ask_user
AI_BASE_URL=ask_user default https://api.openai.com/v1
AI_MODEL_ID=ask_user default gpt-4o-mini
FRONTEND_URL=ask_user

Important:
- Generate JWT_SECRET automatically using openssl rand -hex 32.
- Ask AI_API_KEY securely without echo.
- chmod 600 backend .env.
- Do not print secrets after input.

2. Run:
npm install

3. If Prisma exists:
- npx prisma generate
- npx prisma migrate deploy

4. Build backend:
npm run build

5. Start backend with PM2:
- prefer dist/main.js
- fallback dist/index.js
- fallback npm run start:prod

PM2 name:
ifnote-backend

6. pm2 save

==================================================
FRONTEND INSTALL
==================================================

For frontend installation:

1. Create frontend .env.local.

Required env:

NEXT_PUBLIC_API_BASE_URL="backend API URL"

Ask user:
Backend API URL, example:
https://api.domain.com
or
http://localhost:3001

2. chmod 600 .env.local

3. Run:
npm install
npm run build

4. Start frontend with PM2:

pm2 start npm --name ifnote-frontend -- run start -- -p FRONTEND_PORT

5. pm2 save

==================================================
NGINX SETUP
==================================================

After install, ask:

Mau setup Nginx reverse proxy? y/n

If yes:
- ask frontend domain
- ask backend API domain

Create Nginx config:

server frontend domain:
proxy_pass http://127.0.0.1:FRONTEND_PORT

server backend domain:
proxy_pass http://127.0.0.1:BACKEND_PORT

Add headers:
- Upgrade
- Connection upgrade
- Host
- X-Real-IP
- X-Forwarded-For
- X-Forwarded-Proto

Enable site.
Run:
nginx -t
systemctl reload nginx

Then ask:
Mau install SSL Let's Encrypt sekarang? y/n

If yes:
certbot --nginx -d FRONTEND_DOMAIN -d API_DOMAIN
